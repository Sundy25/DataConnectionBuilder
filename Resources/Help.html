Data Connection Builder Help

What is Data Connection Builder?

Data Connection Builder is used to build connection strings for use with ActiveX Data Objects (ADO), Microsoft's data access technology. An ADO connection string contains the information needed to establish a connection to a data source (such as a database). This information typically includes, at minimum, the name of the data source and the provider. Data Connection Builder currently supports three providers: SQL Server, Oracle, and ACE (used with Access and Excel). A connection string might also include security information, such as user ID and password. For more information on connection strings, see the following resources:
  • ConnectionString Property (Microsoft ADO Programmer's Reference)
  • Connection String Syntax (Microsoft .NET Framework Development Guide)
  • ConnectionStrings.com


To build a new data connection

  1. Specify a Provider and a Data Source. These properties are always required.
  2. Specify other properties as appropriate. Certain properties may be required depending on the provider. Properties that do not apply to the specified provider are hidden.
  3. It is always a good idea to make sure that the connection is valid before saving. Test the connection by clicking the Test Connection button.
